# Astra Product Principles

## Status

Precode foundation document.

## Purpose

This document defines the durable product principles of Astra.

## Principles

### 1. Artifact-first

Astra should prefer durable artifacts over transient conversation.

### 2. Human-accountable

The human remains accountable for consequential decisions.

### 3. Verification-first

Done requires evidence.

### 4. Process-aware

Astra must know which SDLC phase and role mode apply.

### 5. Scrum-compatible

Astra should support Scrum without pretending the AI owns Scrum accountabilities.

### 6. Explicit over implicit

Hidden defaults, silent authority, and undocumented behavior should be avoided.

### 7. Small and reviewable

Work should be sliced into atomic, inspectable changes.

### 8. Traceable

Work should be traceable from goal to evidence.

### 9. Governed

Policies should define boundaries before automation expands.

### 10. Provider-independent

Astra should not be architecturally tied to one LLM provider.
